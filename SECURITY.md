# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this repository, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, email **mikko.parkkola@iki.fi** with:

1. The affected file or section
2. The problematic link, command, or content
3. Steps to reproduce or verify the issue
4. Impact assessment
5. Any suggested fix (optional)

## Security Scope

This repository is primarily documentation and repository metadata.

In scope:

- Malicious or incorrect install commands
- Broken or misleading links to release artifacts or project pages
- Metadata changes that could redirect users to unsafe resources

Out of scope:

- Vulnerabilities in linked tools such as `trvl`, `axterminator`, `mcp-gateway`, `nab`, and `nowifi`
- Security issues in package managers, MCP clients, or local machine configuration

Project-specific vulnerabilities should be reported in the affected repository.
